CAFINITY — PRODUCT PRESENTATION ASSET LIST  (Batch 11)
======================================================
Hot / Iced / Default product imagery. Architecture is live now (product_media
table + resolver); the images below are what to GENERATE so drinks visibly
switch on Hot/Iced. Code is NOT blocked waiting for them — until an image exists
the resolver falls back to the default photo.

HOW IMAGES RESOLVE (lib/productMedia.ts resolveProductImage)
  selected presentation: bundled <slug>-hot/iced.png → remote product_media[hot/iced]
  → default: bundled <slug>.png → remote product_media.default → products.image_url
  → category emoji. Bundled (local) is preferred; admin can also upload remote
  Hot/Iced via the Product Builder (edit mode, when Temperature is attached).

SPECS (per 11.9)
  Path: assets/images/products/   Size: 1000x1000   Format: PNG
  No text / price / badge. Same warm premium café framing as existing photos.
  ICED: clear takeaway cup, visible ice + condensation, recognizable layers.
  HOT : ceramic or hot takeaway cup, steam where appropriate, same recipe identity.
  Filename: <existing-slug>-hot.png  /  <existing-slug>-iced.png
