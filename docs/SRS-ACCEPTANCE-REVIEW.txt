PART A — SRS TEST CASE MAPPING
=====================================================================

TC-01 BRANCH SELECTION ........................................ PASS (code)
- Permission requested ONLY on explicit tap (branches.tsx); states handled:
  undetermined/prompt, granted, denied (re-ask), blocked (Open settings),
  unavailable (device location off / failed → Try again).
- Granted → nearest ACTIVE branch recommended ("Nearest" badge, distance sort).
- Denied/blocked → manual selection still works (list always loads).
- Missing branch coords → no crash (distance chip + badge gated on distanceKm).
- Selecting a branch refreshes inventory (cart re-validates; menu re-queries).
- Active order's branch cannot change (order is immutable post-placement).
Files: app/(app)/branches.tsx, components/ui/BranchSelector.tsx, lib/branchHours.ts

TC-02 OUT-OF-STOCK ITEM ....................................... PASS (code)
- Variant/product availability per branch (branch_inventory, is_available,
  closed-by-default). Sold-out variants disabled on product detail.
- Branch switch is NON-DESTRUCTIVE (rebaseBranch) → cart flags items not
  available at the new branch (red border + per-item message), blocks checkout,
  Edit/Remove offered, no silent deletion. Also catches stock-depletion races.
- Server place_order is the final inventory gate (rejects insufficient stock).
LIMIT: per-CUSTOMIZATION-OPTION availability is NOT modelled in the schema, so it
is not enforced (would need a new column) — documented, not fabricated.
Files: app/(app)/cart.tsx, store/cart.ts, app/(app)/branches.tsx

TC-03 DRINK CUSTOMIZATION ..................................... PASS (code)
- Selections appear in cart + order summary + receipt; add-on prices update
  totals live (product detail summary + sticky bar).
- Required single-choice groups enforced (highlight + scroll to the empty group).
- Historical orders preserve chosen customizations via order_item_customization
  snapshots (and price snapshots — historical totals don't move with menu).
Files: app/(app)/product/[id].tsx, lib/limits.ts

TC-04 PAYMENT SUCCESS ......................................... PASS (code)
- One payment attempt → one order: button disabled while placing + idempotent
  place_order (checkout_request_id).
- Order linked to selected branch; confirmation screen (order/[id]?new=1) with
  pickup number, branch, ETA, payment status, Track.
- Pickup ticket/QR generated (active orders); staff FIFO queue receives it.
- Payment + order statuses separate and correct.
Files: app/(app)/checkout.tsx, app/(app)/order/[id].tsx, components/ui/PickupQR.tsx

TC-05 REWARD POINTS .......................................... PASS (code)
- Points use existing rules (loyalty_points_per_peso); awarded ONCE at COMPLETION
  (advance_order_status), not at placement; pending → earned → reversed lifecycle.
- Balance + activity update; refresh/retry cannot double-award (server-guarded).
Files: supabase/phase5_loyalty_cash.sql (unchanged), rewards.tsx

TC-06 INSUFFICIENT FUNDS ...................................... PASS (code, simulated)
- Simulated GCash balance < total → "Transaction Failed … cart kept"; no order
  created, no points, retry / change method available.
Files: app/(app)/checkout.tsx

TC-07 NETWORK TIMEOUT ......................................... PASS (code)
- Double-submit impossible (disabled while placing + idempotency).
- Offline/timeout on submit → "Verifying your payment" state (NOT a failure):
  "Check payment status" re-calls place_order with the SAME checkout_request_id
  (returns existing order if created, else creates once) → no duplicate.
- Cart preserved; reconnection allows verification.
Files: app/(app)/checkout.tsx, lib/errors.ts, store/network.ts

TC-08 INPUT LIMITS ........................................... PASS (code)
- Quantity capped at MAX_ITEM_QUANTITY (20, shared constant = server check);
  plus button disables at cap + "Maximum of 20 per item." message; value can't
  be set-then-rejected at checkout.
LIMIT: multi-choice MAX-selection not enforced (no max column in schema) — not
fabricated. Required (single) enforced.
Files: lib/limits.ts, components/ui/QuantityStepper.tsx, app/(app)/product/[id].tsx
